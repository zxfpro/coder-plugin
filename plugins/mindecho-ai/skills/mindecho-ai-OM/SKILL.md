# MindEcho AI 部署技能

## 触发条件

当用户请求以下操作时自动触发此技能：
- 部署新版本到测试环境
- 回滚到之前的版本
- 查看测试环境状态
- 查看服务日志
- 重启服务
- 执行健康检查
- 更新 Docker 镜像

## 技能功能

### 1. 自动化部署 (`deploy.sh`)

**功能特性：**
- ✅ 自动构建并推送 Docker 镜像
- ✅ 部署前检查（代码状态、服务器连接）
- ✅ 自动部署到测试环境
- ✅ 健康检查（自动验证服务）
- ✅ 失败时自动回滚
- ✅ 清理旧镜像
- ✅ 彩色日志输出
- ✅ 支持 `-y` 标志跳过交互式确认

**使用方法：**
```bash
# 交互式部署
./deploy.sh

# 非交互式部署（跳过确认）
./deploy.sh -y
```

### 2. 环境回滚 (`rollback.sh`)

**功能特性：**
- ✅ 自动获取可用历史版本
- ✅ 交互式选择回滚目标
- ✅ 只操作测试环境（不涉及生产）
- ✅ 健康检查验证
- ✅ 显示部署日志
- ✅ 支持 `--prev` 快速回滚到上一版本
- ✅ 支持 `-y` 标志跳过交互式确认

**使用方法：**
```bash
# 交互式选择版本回滚
./rollback.sh

# 快速回滚到上一版本
./rollback.sh --prev

# 快速回滚并跳过确认
./rollback.sh --prev -y
```

### 3. 服务状态查看 (`status.sh`)

**功能特性：**
- ✅ 显示当前运行版本
- ✅ 显示容器详细信息
- ✅ 显示资源使用情况
- ✅ 显示可用镜像版本
- ✅ 执行健康检查

**使用方法：**
```bash
./status.sh
```

### 4. 日志查看 (`logs.sh`)

**功能特性：**
- ✅ 显示应用日志
- ✅ 只显示错误日志
- ✅ 显示 Qdrant 服务日志
- ✅ 支持实时日志跟随

**使用方法：**
```bash
# 显示应用日志（默认）
./logs.sh

# 只显示错误日志
./logs.sh --error

# 显示 Qdrant 服务日志
./logs.sh --qdrant

# 实时查看日志
./logs.sh --follow
```

### 5. 服务重启 (`restart.sh`)

**功能特性：**
- ✅ 重启测试服务
- ✅ 重启 Qdrant 服务
- ✅ 重启所有服务
- ✅ 停止/启动服务
- ✅ 自动执行健康检查

**使用方法：**
```bash
# 重启所有服务（默认）
./restart.sh

# 只重启测试服务
./restart.sh --test

# 只重启 Qdrant 服务
./restart.sh --qdrant

# 停止所有服务
./restart.sh --stop

# 启动所有服务
./restart.sh --start
```

### 6. 健康检查 (`health.sh`)

**功能特性：**
- ✅ 检查测试环境健康状态
- ✅ 检查 Qdrant 服务健康
- ✅ 检查数据库连接
- ✅ 显示响应时间
- ✅ 支持快速检查和详细检查

**使用方法：**
```bash
# 详细健康检查（默认）
./health.sh

# 快速健康检查
./health.sh --quick
```

### 7. 配置管理 (`config.sh`)

**功能特性：**
- ✅ 集中管理所有配置变量
- ✅ 提供统一的颜色输出函数
- ✅ 易于维护和更新配置

**配置项：**
- 服务器地址、用户、目录
- 测试/生产环境端口
- Docker 镜像仓库
- 健康检查参数
- 日志参数

## 部署流程

### 标准部署流程

1. **代码提交** - 确保所有更改已提交
2. **构建镜像** - `docker buildx build --platform linux/amd64`
3. **推送镜像** - 推送到阿里云仓库
4. **拉取镜像** - 服务器拉取最新镜像
5. **部署容器** - `docker-compose up -d --force-recreate`
6. **健康检查** - 验证 HTTP 200 响应

### 回滚流程

1. **查看历史版本** - 显示可用镜像列表
2. **选择版本** - 交互式选择目标版本
3. **停止容器** - `docker-compose down`
4. **启动容器** - 使用 `IMAGE_TAG` 环境变量启动
5. **健康检查** - 验证服务正常运行

## 注意事项

⚠️ **安全限制：**
- 此技能主要针对测试环境部署
- 生产环境蓝绿部署由其他流程控制
- 此技能仅提供生产环境的监控和状态查看
- 所有操作需要 SSH 密钥认证

⚠️ **回滚限制：**
- 只能回滚到服务器上存在的镜像
- 建议在业务验证失败时及时回滚
- 回滚不会删除旧镜像，便于多次回滚

## 常用运维场景

### 场景1：部署新功能
```bash

# 2. 部署到测试环境
./deploy.sh

# 3. 验证功能
./status.sh

```

### 场景2：快速回滚
```bash

./rollback.sh --prev -y
```

```bash
# 1. 查看服务状态
./status.sh

# 2. 查看错误日志
./logs.sh --error

# 3. 如果需要，重启服务
./restart.sh --test

./health.sh --detailed
```

### 场景4：监控服务
```bash

./logs.sh --follow

# 2. 另一个终端查看状态
./status.sh

watch -n 10 ./health.sh --quick
```
